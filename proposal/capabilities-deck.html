Understanding the problem

Where Meridian is today

What's working

A Vue/Python dashboard covering core inventory, orders, and spending views — used daily by your operations team across three warehouses.

What isn't

Reports module unfinished at handoff. No test coverage — blocking IT sign-off on any changes. Tokyo team working in English-only views. No Restocking capability.

The real problem

The previous vendor handed off an incomplete system with minimal documentation. Every change carries risk because there's nothing to catch regressions. Your team is stuck.

Codebase review

What the handoff documentation tells us

  • The vendor's own notes confirm the Reports module was "in progress" at contract end — this is remediation of incomplete work, not routine bug fixing
  • No automated tests were delivered — IT's gatekeeper requirement for future changes
  • Frontend migration from Options API to Composition API was started but not finished
  • Handoff documentation is four bullet points — we will conduct our own architecture review before committing to fixed estimates

Requirement R4

Architecture review

  • Independent audit — we verify rather than trust the previous vendor's notes
  • Covers frontend (Vue 3 components, routing, API client), backend (FastAPI routes, data layer), and gaps
  • Deliverable: HTML diagram + written narrative suitable for handoff to Meridian IT
  • Produced in Week 2 — before any fix work begins, findings inform our R1 scope
Requirement R1

Reports module remediation

  • Systematic audit across all four filter dimensions — Time Period, Warehouse, Category, Order Status — and their API interactions
  • Every defect documented before any fixes are applied; Meridian sees the full list
  • Fix order: filter wiring → i18n gaps → API inconsistencies → console noise
  • Assumption: no existing bug log provided; audit is in-scope. If you share one, we reconcile at kickoff.
Requirement R3

Automated browser testing

  • End-to-end tests using Playwright — the industry standard for web application coverage
  • Coverage: inventory browsing, order views, Reports page, Restocking view
  • Sequenced before R2 so the Restocking feature ships with tests from day one
  • Structured for CI integration — Meridian IT can run these in any future pipeline
Requirement R2

Restocking recommendations

Inputs

Current stock levels, demand forecast (both from existing APIs), and an operator-supplied budget ceiling entered in the UI.

Logic

New FastAPI endpoint ranks items by shortfall severity × demand urgency, filtered to fit within the budget ceiling.

Output

Ranked purchase order recommendations displayed in a new Vue view — no external system integration required in Phase 1.
