Findings & Known Issues

SeverityAreaIssue
High Reports.vue Uses Options API while the entire rest of the codebase uses Composition API. Filters not wired up — previous vendor left this incomplete.
High Backlog API Backend returns has_purchase_order (boolean), but frontend checks item.purchase_order_id. Conditional rendering on Dashboard is broken.
High Purchase Orders POST /api/purchase-orders and GET /api/purchase-orders/{id} not implemented. PurchaseOrderModal is non-functional.
Medium Spending API All /api/spending/* endpoints return static data — warehouse/category/period filters have no effect on spending views.
Medium Reports filters Reports page filter integration incomplete — some or all of the 4 filters do not trigger data reloads.
Medium Tasks API /api/tasks/* endpoints not implemented. App.vue loads tasks on mount and fails silently (console.error only).
Low i18n Some views/modules not covered by i18n — Tokyo team sees English-only content in certain screens.
Low Quarter mapping Q-format quarter mapping is hardcoded to 2025. Will fail silently for other years.
Low No tests Zero automated test coverage — no unit, integration, or end-to-end tests delivered.
